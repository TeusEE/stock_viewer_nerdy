# PRD (Product Requirements Document)

> 제품: **주식 관심종목 (Stock Watch)**
> 버전: 1.0 / 최종 수정일: 2026-05-29

---

## 1. 개요 (Overview)

한국 주식시장(KRX) 종목을 **종목명으로 검색**해 관심종목으로 등록하고, 터미널에서
**현재가·등락률·고가·저가**를 표로 확인하며 **실시간 감시(자동 갱신)** 할 수 있는
인터랙티브 CLI/TUI 애플리케이션.

- 데이터: `FinanceDataReader`
- 화면: `rich` 기반 컬러 TUI
- 실행: `run.bat` (Windows) 또는 `python app\main.py`

---

## 2. 배경 / 문제 정의 (Problem)

- 개인 투자자가 가벼운 도구로 관심종목 시세를 빠르게 보고 싶어 한다.
- 종목코드(예: `005930`)를 외우기 어렵다 → **이름으로 찾고 이름으로 보고 싶다.**
- 매번 명령을 따로 입력(`add`, `list` 등)하는 방식은 번거롭다 → **한 화면 안에서
  관리**하고 싶다.

---

## 3. 목표 (Goals) / 비목표 (Non-Goals)

### 목표
- G1. 종목명 검색 → 후보 추천 → 선택 등록의 직관적 흐름 제공
- G2. 관심종목 추가/수정/삭제를 프로그램 화면 안에서 수행
- G3. 시세 표를 종목명 중심으로, 한국 시장 관례에 맞게 표시
- G4. 자동 갱신 감시 모드 + 사용자가 갱신 주기 설정 가능
- G5. 관심종목/설정의 영속 저장(재실행 유지)

### 비목표
- N1. 실제 매매/주문, 증권사 계좌 연동
- N2. 해외 주식, 초 단위 실시간 틱 데이터
- N3. 다중 사용자, 클라우드 동기화
- N4. 그래프/차트 시각화 (향후 확장 후보)

## 4. 대상 사용자 (Target Users)

| 페르소나 | 설명 | 핵심 니즈 |
| -------- | ---- | --------- |
| 개인 투자자(한국) | 소수의 관심종목을 가볍게 추적 | 빠른 조회, 이름 기반 검색 |
| 터미널 친화 사용자 | CLI 환경을 선호하는 개발자/파워유저 | 키보드만으로 빠른 조작 |

---

## 5. 기능 요구사항 (Functional Requirements)

| ID | 기능 | 설명 | 우선순위 |
| -- | ---- | ---- | -------- |
| FR-1 | 종목 검색 | 종목명 부분일치로 후보 목록(최대 20) 표시, 정렬(완전>접두>부분) | High |
| FR-2 | 관심종목 추가 | 검색 후보에서 번호 선택 시 (코드,이름) 등록, 중복 방지 | High |
| FR-3 | 관심종목 삭제 | 표의 번호 선택으로 제거 | High |
| FR-4 | 관심종목 수정 | 기존 항목을 새로 검색한 종목으로 교체(위치 유지) | High |
| FR-5 | 관심종목 조회 | 종목명/코드/현재가/등락률/고가/저가 표 출력 | High |
| FR-6 | 시세 새로고침 | 전체 관심종목 시세 재조회 | Medium |
| FR-7 | 실시간 감시 | 설정된 주기로 자동 갱신, Ctrl+C로 메뉴 복귀 | High |
| FR-8 | 갱신 주기 설정 | 설정 메뉴에서 주기(초) 변경, 1~3600 클램핑, 영속 저장 | High |
| FR-9 | 영속 저장 | 관심종목/설정을 JSON으로 저장·로드 | High |
| FR-10 | 종목 목록 캐시 | KRX 목록을 당일 1회 다운로드 후 로컬 CSV 캐시 | Medium |

## 6. 비기능 요구사항 (Non-Functional Requirements)

| ID | 항목 | 요구사항 |
| -- | ---- | -------- |
| NFR-1 | 호환성 | Windows CMD/PowerShell/Windows Terminal, Python 3.11+ |
| NFR-2 | 한글 출력 | UTF-8 강제 설정으로 한글/박스문자 깨짐 방지(cp949 대응) |
| NFR-3 | 안정성 | 네트워크/데이터 오류 시 크래시 없이 친절한 메시지 |
| NFR-4 | 응답성 | 메뉴 이동 시 캐시된 시세 사용, 변경 시에만 네트워크 호출 |
| NFR-5 | 가독성 | 종목명 우선, 원화 콤마 표기, 상승=빨강/하락=파랑 |
| NFR-6 | 단순 배포 | PyInstaller 단일 EXE 빌드 가능 |
| NFR-7 | 데이터 보존 | 잘못된 설정/손상 파일에도 기본값으로 복구 |

## 7. 데이터 사양 (Data)

### `data/watchlist.json`
```json
{
  "watchlist": [
    { "code": "005930", "name": "삼성전자" },
    { "code": "035720", "name": "카카오" }
  ]
}
```

### `data/settings.json`
```json
{ "watch_interval": 5 }
```

### `data/krx_listing.csv` (자동 생성/캐시)
`code, name, market` 컬럼, 당일 자정 기준 재다운로드.

## 8. 오류 처리 (Error Handling)

| 상황 | 메시지 |
| ---- | ------ |
| 네트워크/데이터 실패 | `[ERROR] 종목 정보를 불러오지 못했습니다.` |
| 검색 결과 없음 | `[INFO] '...' 와(과) 일치하는 종목이 없습니다.` |
| 중복 추가 | `[INFO] 이미 등록된 종목입니다.` |
| 빈 관심종목 | `[INFO] 관심 종목이 비어 있습니다.` |
| 잘못된 입력(숫자) | `숫자를 입력하세요.` / 범위 안내 후 재입력 |

---

## 9. 성공 지표 (Success Metrics)

- 종목명만 입력해도 5초 이내에 원하는 종목을 찾아 등록할 수 있다.
- 관심종목 등록/삭제/수정이 명령 재입력 없이 한 화면에서 완료된다.
- 감시 모드에서 사용자가 원하는 주기로 시세가 자동 갱신된다.
