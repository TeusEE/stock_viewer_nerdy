# 도메인 정의서 (Domain Definition)

> 프로젝트: **주식 관심종목 (Stock Watch)** — 한국 주식시장(KRX) 전용 CLI/TUI
> 최종 수정일: 2026-05-29

---

## 1. 목적

본 문서는 "주식 관심종목" 애플리케이션이 다루는 **업무 도메인의 핵심 개념, 용어,
규칙, 경계**를 정의한다. 개발/기획 참여자가 동일한 어휘로 소통하기 위한 기준이다.

---

## 2. 도메인 범위 (Scope)

| 구분 | 내용 |
| ---- | ---- |
| 대상 시장 | 한국거래소(KRX) — KOSPI, KOSDAQ, KONEX |
| 사용 환경 | 한국 내 개인 사용자, Windows 터미널(CMD/PowerShell/Windows Terminal) |
| 핵심 행위 | 종목 검색, 관심종목 관리(추가/수정/삭제), 시세 조회, 실시간 감시 |
| 데이터 출처 | `FinanceDataReader` (KRX 종목 목록 + 일봉 시세) |
| 범위 외 | 실제 매매/주문, 계좌 연동, 해외 주식, 초 단위 틱 데이터, 다중 사용자 |

---

## 3. 핵심 용어 (Ubiquitous Language)

| 용어 | 영문/식별자 | 정의 |
| ---- | ----------- | ---- |
| 종목 | Stock | 거래소에 상장된 개별 회사의 주식 |
| 종목코드 | Code | 6자리 KRX 종목 식별자 (예: `005930`) |
| 종목명 | Name | 사람이 읽는 회사 이름 (예: `삼성전자`) |
| 시장 | Market | 종목이 속한 시장 구분 (KOSPI / KOSDAQ / KONEX) |
| 관심종목 | Watchlist | 사용자가 추적하기 위해 등록한 종목들의 집합 |
| 관심종목 항목 | Watchlist Entry | 관심종목을 구성하는 단위 = (종목코드, 종목명) |
| 시세 | Quote | 특정 종목의 가격 정보 (현재가/등락률/고가/저가) |
| 현재가 | Price | 최신 일봉의 종가(Close) |
| 등락률 | Change % | 전일 종가 대비 가격 변동 비율 |
| 고가 / 저가 | High / Low | 당일 최고가 / 최저가 |
| 감시 | Watch | 관심종목 시세를 일정 주기로 자동 갱신해 보여주는 모드 |
| 갱신 주기 | Watch Interval | 감시 모드에서 화면을 다시 그리는 시간 간격(초) |
| 종목 목록 | Listing | KRX 전체 상장 종목의 (코드, 이름, 시장) 목록 |

---

## 4. 핵심 엔티티 (Entities)

### 4.1 Stock (종목)
- 속성: `code`(PK), `name`, `market`
- KRX 종목 목록(Listing)으로부터 파생되는 읽기 전용 개념

### 4.2 WatchlistEntry (관심종목 항목)
- 속성: `code`(식별자), `name`
- 규칙:
  - `code`는 관심종목 내에서 **유일**해야 한다 (중복 등록 불가)
  - `name`은 등록 시점의 종목명을 함께 저장하여 오프라인에서도 표시 가능

### 4.3 Quote (시세)
- 속성: `code`, `price`, `change_percent`, `high`, `low`
- 규칙: 데이터 조회 실패 시 각 값은 `None`(표시상 `N/A`)일 수 있다

### 4.4 Settings (설정)
- 속성: `watch_interval` (정수, 초)
- 규칙: `1 ~ 3600` 범위로 강제(클램핑), 기본값 `5`

---

## 5. 도메인 규칙 (Business Rules)

1. **중복 금지** — 동일 `code`는 관심종목에 한 번만 존재한다.
2. **종목명 우선 표시** — 모든 화면에서 코드가 아닌 종목명을 우선 노출한다.
3. **한국 시장 색상 관례** — 상승은 빨강, 하락은 파랑, 보합/없음은 흐림(dim).
4. **원화 표기** — 가격은 정수 + 천 단위 콤마(예: `317,000`), 소수점 없음.
5. **검색 기준** — KRX에 등록된 종목명 부분일치로 검색하며, 완전일치 > 접두사
   일치 > 부분일치 순으로 정렬한다(최대 20건).
6. **영문 등록 예외** — 일부 종목은 KRX에 영문명으로 등록(예: `NAVER`)되어
   한글 검색이 되지 않을 수 있다(데이터 출처의 한계).
7. **지속성** — 관심종목과 설정은 프로그램 종료 후에도 유지된다.
8. **설정 클램핑** — 갱신 주기는 허용 범위를 벗어나면 자동으로 조정 저장한다.

## 6. 데이터 / 외부 경계 (Boundaries)

| 경계 | 설명 |
| ---- | ---- |
| 시세 데이터 | `FinanceDataReader.DataReader(code)` — **일봉(EOD) 기준**, 실시간 틱 아님 |
| 종목 목록 | `FinanceDataReader.StockListing('KRX')` — 일 1회 다운로드 후 로컬 캐시 |
| 영속 저장 | 로컬 JSON 파일 (`data/watchlist.json`, `data/settings.json`) |
| 목록 캐시 | 로컬 CSV (`data/krx_listing.csv`, 당일 캐시) |

---

## 7. 도메인 한계 및 가정 (Assumptions / Limitations)

- 시세는 **지연된 일봉 데이터**이므로 감시 모드가 초 단위로 가격을 바꾸지 않는다.
  ("화면 갱신 주기"와 "데이터 신선도"는 별개 개념)
- 단일 사용자, 단일 머신(로컬 파일) 기준. 동시성/동기화는 고려하지 않는다.
- 네트워크가 없으면 신규 검색/시세 조회는 불가하나, 저장된 관심종목 명칭은
  표시 가능하다.
