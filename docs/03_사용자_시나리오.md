# 사용자 시나리오 (User Scenarios)

> 제품: **주식 관심종목 (Stock Watch)**
> 최종 수정일: 2026-05-29 (뉴스 기능 시나리오 추가)

본 문서는 실제 구현된 동작을 기준으로 주요 사용 흐름을 시나리오 형태로 기술한다.

---

## 페르소나

- **지민 (개인 투자자)**: 삼성전자·카카오 등 몇 개 종목을 가볍게 추적하고 싶다.
  종목코드는 잘 모르고, 회사 이름으로 찾고 싶어 한다. 터미널 사용에 익숙하다.

---

## 시나리오 1. 앱 실행 및 첫 화면

**전제** 관심종목이 비어 있음
1. 지민이 `run.bat` 을 실행한다.
2. 상단 타이틀(`주식 관심종목`)과 함께 `[INFO] 관심 종목이 비어 있습니다.` 가 표시된다.
3. 하단 메뉴가 나타난다: `[a] 추가 [e] 수정 [d] 삭제 [w] 감시 [r] 새로고침 [s] 설정 [q] 종료`

**결과** 사용자는 다음 행동을 선택할 수 있다.

---

## 시나리오 2. 종목명으로 검색하여 추가 (핵심 흐름)

**전제** 앱 실행 중
1. 지민이 `a` 를 입력한다.
2. "추가할 종목명을 입력하세요 (예: 삼성)" 프롬프트에 `삼성` 을 입력한다.
3. 유사 종목 후보가 번호와 함께 표로 표시된다 (종목명/코드/시장).
   ```
   [1] 삼성전자       005930  KOSPI
   [2] 삼성전자우     005935  KOSPI
   [3] 삼성SDI        006400  KOSPI
   ...
   [0] 취소
   ```
4. 지민이 `1` 을 선택한다.
5. `[INFO] 삼성전자 추가 완료.` 메시지가 뜨고 Enter로 메인 화면 복귀.
6. 관심종목 표에 `삼성전자` 가 현재가/등락률과 함께 표시된다.

**예외**
- 일치 결과 없음 → `[INFO] '...' 와(과) 일치하는 종목이 없습니다.`
- 이미 등록됨 → `[INFO] 이미 등록된 종목입니다.`
- `0` 입력 또는 빈 입력 → 취소하고 메뉴로 복귀

---

## 시나리오 3. 실시간 감시 모드 (뉴스 포함)

**전제** 관심종목에 1개 이상 등록됨
1. 지민이 `w` 를 입력한다.
2. "실시간 감시" 화면으로 전환되어 종목명 기준 시세 표가 표시된다.
3. 설정된 갱신 주기(기본 5초)마다 시세 표가 자동으로 다시 그려진다.
4. 시세 표 아래 구분선이 표시되고, 최초 진입 시 백그라운드에서 뉴스 파싱이 시작된다.
5. 뉴스가 로드되면 구분선 아래에 최대 5건의 뉴스가 `제목 | 출처 | 링크` 형태로 표시된다.
6. 뉴스는 5분(300초)마다 자동으로 재파싱되어 최신 기사로 교체된다.
7. 하단에 `{N}초마다 갱신됩니다. 뉴스 {M}분 후 갱신. Ctrl+C 로 복귀.` 안내.
8. 지민이 `Ctrl+C` 를 누르면 메인 메뉴로 복귀한다.

**비고**
- 상승은 빨강, 하락은 파랑으로 등락률이 표시된다(한국 시장 관례).
- 뉴스 파싱 중에는 이전 뉴스가 그대로 유지되어 화면이 빈 상태로 보이지 않는다.
- 뉴스 파싱은 시세 갱신을 차단하지 않는다.

**예외**
- 관심종목이 비어 있으면 `[INFO] 관심 종목이 비어 있습니다.` 후 복귀.
- 뉴스 파싱에 전부 실패하면 뉴스 섹션이 생략된다(시세 표는 정상 동작).
- 개별 소스 실패 시 나머지 소스의 결과만 표시한다.

## 시나리오 4. 설정 변경

**전제** 앱 실행 중
1. 지민이 `s` 를 입력한다.
2. 3개 항목과 현재값이 번호와 함께 표시된다:
   ```
   설정

     [1] 시세 갱신 주기       5초  (1 ~ 3600초)
     [2] 뉴스 갱신 주기     300초  (60 ~ 3600초)
     [3] 최대 뉴스 개수       5건  (1 ~ 10건)

     [0] 취소
   ```
3. 지민이 변경할 항목 번호를 입력한다.
4. 현재값과 함께 새 값 입력 프롬프트가 나타난다.
5. 새 값을 입력하면 `[INFO] ... 저장했습니다.` 메시지 후 메뉴 복귀.
6. 해당 설정은 `data/settings.json` 에 즉시 반영된다.

**예시 — 시세 갱신 주기를 10초로 변경**
- `1` 선택 → `새 시세 갱신 주기(초): 10` → `[INFO] 시세 갱신 주기을(를) 10초로 저장했습니다.`

**예시 — 뉴스 갱신 주기를 10분으로 변경**
- `2` 선택 → `새 뉴스 갱신 주기(초): 600` → `[INFO] 뉴스 갱신 주기을(를) 600초로 저장했습니다.`
- 이후 감시 모드의 뉴스는 10분 주기로 갱신된다.

**예시 — 최대 뉴스 개수를 3건으로 변경**
- `3` 선택 → `새 최대 뉴스 개수(건): 3` → `[INFO] 최대 뉴스 개수을(를) 3건으로 저장했습니다.`

**예외**
- 범위 밖 값(예: `9999`) → 자동 조정 후 `[INFO] 허용 범위로 조정되어 {최댓값}으로 저장했습니다.`
- 숫자가 아니면 → `숫자를 입력하세요.`
- 빈 입력 또는 `0` → 변경 없이 메뉴 복귀

## 시나리오 5. 관심종목 수정

**전제** 관심종목에 `삼성전자` 등록됨
1. 지민이 `e` 를 입력한다.
2. 번호가 매겨진 관심종목 표에서 수정할 항목 번호(`1`)를 선택한다.
3. "'삼성전자' 대신 등록할 종목명 입력" 프롬프트에 `카카오` 를 입력한다.
4. 검색 후보에서 `카카오` 를 선택한다.
5. `[INFO] 삼성전자 → 카카오 수정 완료.` 후 위치를 유지한 채 교체된다.

**예외** 새 검색 결과가 없거나 취소하면 수정이 일어나지 않는다.

---

## 시나리오 6. 관심종목 삭제

**전제** 관심종목에 1개 이상 등록됨
1. 지민이 `d` 를 입력한다.
2. 관심종목 표에서 삭제할 번호를 선택한다.
3. `[INFO] {종목명} 삭제 완료.` 후 해당 항목이 사라진다.

**예외** `0` 입력 시 취소.

---

## 시나리오 7. 시세 새로고침 / 종료

- `r` : 전체 관심종목의 최신 시세를 다시 불러와 표를 갱신한다.
- `q` : `종료합니다.` 메시지 후 앱을 종료한다. 관심종목/설정은 그대로 보존된다.

---

## 시나리오 8. 뉴스 자동 갱신

**전제** 감시 모드 진입 후 5분 경과
1. 백그라운드 스레드가 5분 간격으로 Naver Finance, Yahoo Finance, Bloomberg를
   순서대로 파싱한다.
2. 새 뉴스 데이터가 캐시에 반영되면 다음 화면 렌더링 시 자동으로 업데이트된다.
3. 하단 안내문의 "뉴스 N분 후 갱신" 카운트다운이 0이 되면 재파싱이 시작된다.

**예외**
- 5분 주기 재파싱 도중 소스 접속 실패 → 해당 소스 결과 생략, 기존 캐시 유지.
- 모든 소스 실패 → 기존 캐시 유지(화면 변화 없음).

## 예외/엣지 케이스 요약

| 케이스 | 동작 |
| ------ | ---- |
| 네트워크 단절 | 시세 `N/A` 표시, `[ERROR] 종목 정보를 불러오지 못했습니다.` |
| 영문 등록 종목(NAVER 등) | 한글명으로 검색 시 결과 없음 (코드/영문명으로 검색 필요) |
| 손상된 설정 파일 | 각 항목 기본값(시세 5초 / 뉴스 300초 / 최대 5건)으로 복구 |
| 잘못된 메뉴 키 | rich 프롬프트가 유효한 선택만 허용 |
| 뉴스 소스 전체 실패 | 뉴스 섹션 생략, 시세 감시는 정상 동작 |
| 뉴스 소스 일부 실패 | 성공한 소스의 결과만 표시 |
| 뉴스 파싱 중 Ctrl+C | 파싱 스레드 종료 후 메뉴 복귀 |
